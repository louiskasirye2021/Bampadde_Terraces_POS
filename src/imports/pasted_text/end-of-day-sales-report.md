You are generating a beautifully designed, print-ready End of Day Sales Report 
for a Point of Sale (POS) system. The report must be visually rich, data-dense, 
and professionally structured. Use a modern dashboard-style layout with charts, 
graphs, summary cards, and well-organized tables.

---

## DESIGN REQUIREMENTS

- Clean, modern aesthetic with a professional color palette
  (suggested: deep navy #1a2b4a as primary, gold/amber #f5a623 as accent, 
  white backgrounds, soft gray section dividers)
- Business logo and name prominently displayed at the top
- Clear visual hierarchy: large KPI cards → charts → detailed tables
- Each section separated by styled headers with icons
- Fully responsive layout that prints cleanly on A4/Letter paper
- Page numbers and generation timestamp in the footer
- All monetary values formatted with currency symbol and 2 decimal places
- Color coding throughout:
    - Green (#27ae60) for income, revenue, profit
    - Red (#e74c3c) for expenses, voids, losses
    - Blue (#2980b9) for neutral data and quantities
    - Amber (#f39c12) for warnings (low stock, high expenses)

---

## REPORT STRUCTURE

### HEADER SECTION
- Business name, logo placeholder, and tagline
- Report title: "End of Day Sales Report"
- Session date, start time, end time, and total hours of operation
- Generated by (staff name) and report generation timestamp

---

### SECTION 1 — KEY PERFORMANCE INDICATORS (KPI Cards)
Display as a row of large, bold summary cards with icons:

| Card | Value | Icon |
|------|-------|------|
| Total Sales (Orders) | Number of completed transactions | 🧾 |
| Total Income | Sum of all completed payments | 💰 |
| Total Expenses | Sum of all logged expenses | 📉 |
| Net Revenue | Total Income minus Total Expenses | 📊 |
| Average Order Value | Total Income ÷ Total Sales | 🛒 |
| Items Sold | Total quantity of all items sold today | 📦 |

Each card should have:
- A colored top border (green for income, red for expenses, blue for others)
- A bold large number
- A small label underneath
- A subtle icon or emoji

---

### SECTION 2 — REVENUE BREAKDOWN (Charts)
Include TWO visual charts side by side:

**Chart A — Sales by Payment Method (Pie or Donut Chart)**
- Segments for: Cash, Card, Mobile Money, Credit, Other
- Show percentage and value per segment
- Legend with color swatches

**Chart B — Sales Over Time (Bar or Line Chart)**
- X-axis: hours of the day (e.g., 8AM, 9AM ... 10PM)
- Y-axis: revenue or number of orders per hour
- Helps visualize peak and slow periods during the day

Use Chart.js or a similar charting library to render these charts.

---

### SECTION 3 — TOTAL SALES SUMMARY
A clean structured table with these columns:

- Alternating row colors for readability
- Status badges: Completed (green), Voided (red), Refunded (amber)

A detailed table of all expenses recorded during the day:

- Total expenses row at the bottom (bold, red)

### SECTION 5 — ITEMS SOLD: MOST TO LEAST (Sales Ranking)
Display a horizontal bar chart AND a detailed table below it:

**Horizontal Bar Chart:**
- Each bar represents an item
- Bar length = quantity sold
- Sorted from most sold (top) to least sold (bottom)
- Color gradient from green (top sellers) to gray (low sellers)
- Value labels on each bar

**Detailed Table (below the chart):**

| Rank | Item Name | Category | Unit Price | Qty Sold | Total Revenue | % of Sales |
|------|-----------|----------|------------|----------|---------------|------------|

- Top 3 items highlighted with gold/silver/bronze badges 🥇🥈🥉
- Show percentage contribution to total revenue
- A "% of Sales" mini bar within the table cell (inline visual)

Include a mini summary above the table:

## TECHNICAL IMPLEMENTATION NOTES

- Render the report as an HTML page styled with CSS (no frameworks needed, 
  use pure CSS or Tailwind)
- Use Chart.js (CDN) for all charts — pie, bar, horizontal bar, line
- Include a "Download as PDF" button that uses window.print() with 
  print-specific CSS, or jsPDF + html2canvas for true PDF export
- PDF filename format: EndOfDayReport_YYYY-MM-DD.pdf
- All data should be passed in as a single JavaScript data object at the 
  top of the file so it is easy to connect to a real backend later
- Include realistic dummy/sample data so the report renders fully on first load
- Make the report printable: hide buttons, adjust margins, ensure no content 
  is cut off across page breaks
- The report should work as a single self-contained HTML file
